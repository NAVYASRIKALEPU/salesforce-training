# Profiles, Permissions, and Access Control

## Profiles and Permissions

* Profiles control what users can do in Salesforce.
* Every user must have one profile.
* Permissions define access to objects, fields, tabs, and records.
* Profiles help manage user roles and responsibilities.
* Permission Sets provide extra permissions without changing profiles.

Examples:

* Read, Create, Edit, Delete permissions
* App and tab visibility
* Apex class access

---

## Enterprise Security Basics

* Security protects business data from unauthorized access.
* Salesforce uses multiple security layers.
* Authentication verifies user identity.
* Authorization controls what users can access.
* Data security ensures privacy and protection.
* Strong security improves system reliability and trust.

---

## Access Control

* Access control determines who can view or modify data.
* Salesforce uses:

  * Organization-Level Security
  * Object-Level Security
  * Field-Level Security
  * Record-Level Security
* Sharing rules and roles help manage data access.
* Proper access control prevents data misuse and maintains confidentiality.
