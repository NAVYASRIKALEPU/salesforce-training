# GitHub Workflow, CI/CD Concepts, Salesforce Deployment Pipeline, and DevOps Mindset

## GitHub Workflow

* GitHub workflow helps teams manage and collaborate on code efficiently.
* Developers use repositories to store and track project files.
* Branches are used for developing new features separately.
* Pull requests help review and merge code safely.
* GitHub improves collaboration and version control.

### Common Workflow

1. Clone repository
2. Create feature branch
3. Develop and test code
4. Commit changes
5. Push to GitHub
6. Create pull request
7. Review and merge code

---

# CI/CD Concepts

* CI/CD stands for Continuous Integration and Continuous Deployment.
* Continuous Integration automatically tests and validates code changes.
* Continuous Deployment automatically releases validated code to environments.
* Helps deliver updates faster and more reliably.

### Benefits

* Faster deployments
* Reduced manual work
* Early bug detection
* Improved software quality
* Better team collaboration

### Common CI/CD Tools

* GitHub Actions
* Jenkins
* GitLab CI
* Azure DevOps

---

# Salesforce Deployment Pipeline

* Deployment pipeline is the process of moving code between environments.
* Salesforce uses sandboxes, scratch orgs, and production orgs.
* Automated testing ensures deployment quality.
* Pipelines improve release management and reliability.

### Pipeline Stages

1. Development
2. Testing
3. Staging
4. Production Deployment

### Deployment Tools

* Salesforce CLI
* Change Sets
* DevOps Center
* Metadata API

---

# DevOps Mindset

* DevOps combines development and operations practices.
* Focuses on automation, collaboration, and continuous improvement.
* Encourages faster and reliable software delivery.
* Helps teams maintain stable enterprise applications.

### Key Principles

* Automation
* Collaboration
* Continuous Testing
* Continuous Monitoring
* Faster Delivery

### Advantages

* Improved productivity
* Better deployment reliability
* Reduced errors
* Faster release cycles
* Enhanced application quality
