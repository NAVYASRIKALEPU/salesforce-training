# Objects and Fields in Salesforce
Salesforce stores and manages data in a structured format similar to an Excel spreadsheet. It uses Objects, Fields, and Records to organize information properly.

## Objects
Objects are like tables in a database. They are used to store a specific type of information. Salesforce provides standard objects such as Account, Lead, Contact, and Opportunity. Users can also create custom objects like Employee, Student, or Library depending on business needs.

## Fields
Fields are the columns inside an object. They store specific details about the data. For example, an Employee object may contain fields such as Name, Email, Phone Number, Address, and Salary. Different field types can store text, numbers, dates, and more.

## Records
Records are the rows inside an object. Each record contains complete information about one entry. For example, the details of one employee or one customer represent a single record.

## Schema Builder
Schema Builder is a visual tool in Salesforce that helps users view objects, fields, and relationships between objects. It provides an easy drag-and-drop interface to understand the database structure.

## Object Manager
Object Manager is used to create and manage objects and fields in Salesforce. It allows users to customize the platform by adding new fields, editing existing ones, and managing object settings.

## Example
If there is an Employee object:
* Object = Employee
* Fields = Name, Email, Salary
* Record = Details of one employee

This structure helps Salesforce organize and manage business data efficiently.
