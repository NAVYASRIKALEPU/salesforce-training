# Flow Builder Basics, Flow Structure, Resources, and Logic

## Flow Builder Basics

* Flow Builder is a Salesforce tool used to automate business processes.
* It provides a visual interface for creating workflows without heavy coding.
* Flows help automate tasks, data updates, approvals, and user interactions.
* Used for improving productivity and reducing manual work.

### Types of Flows

* Screen Flow
* Record-Triggered Flow
* Scheduled Flow
* Autolaunched Flow

### Benefits

* Low-code automation
* Faster business processes
* Improved efficiency
* Reduced manual errors

---

# Flow Structure

* A flow is made up of different elements connected together.
* Each element performs a specific action in the automation process.

### Main Flow Elements

1. **Start**

   * Defines how the flow begins.

2. **Assignment**

   * Assigns values to variables.

3. **Decision**

   * Adds conditions and branching logic.

4. **Get Records**

   * Retrieves Salesforce records.

5. **Create/Update/Delete Records**

   * Performs database operations.

6. **Screen**

   * Displays UI for user input.

7. **End**

   * Stops the flow execution.

### Basic Flow Structure

Start → Logic → Action → End

# Resources and Logic

* Resources store and manage data inside flows.
* Logic controls decision-making and automation behavior.

### Types of Resources

* Variables
* Constants
* Formulas
* Collections

### Logic Components

* Decision elements
* Loops
* Conditions
* Formula evaluations

### Example

* If Account Type = Customer → Send Email
* Else → Update Record
